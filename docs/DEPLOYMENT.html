⚡ ElectroShop — Ghid de deployment gratuit

Pui aplicația online folosind trei servicii gratuite. Datele (utilizatori, rol admin, comenzi) stau într-o bază de date MySQL persistentă.

Arhitectura online
VercelFrontend React (gratuit permanent)
RenderBackend Spring Boot (gratuit, 750h/lună)
AivenBază de date MySQL (gratuit „forever")

Flux: browserul încarcă frontend-ul de pe Vercel → acesta apelează API-ul de pe Render → care citește/scrie în MySQL pe Aiven.

Înainte de start — creează 4 conturi gratuite (cu email sau „Sign in with GitHub"): github.com, aiven.io, render.com, vercel.com.

1 Pune codul pe GitHub

Render și Vercel iau codul dintr-un repository GitHub. Dezarhivează electroshop.zip, apoi din folderul electroshop:

git init
git add .
git commit -m "ElectroShop initial"
# creează un repo gol pe github.com (ex: electroshop), apoi:
git branch -M main
git remote add origin https://github.com/<utilizatorul-tău>/electroshop.git
git push -u origin main

Alternativ, fără linia de comandă: pe github.com apeși Add file → Upload files și tragi tot folderul.

2 Creează baza de date MySQL pe Aiven

  1. Pe aiven.io → Create service → MySQL.
  2. Alege planul Free, o regiune apropiată (ex. Europe), nume electroshop-db → Create.
  3. Așteaptă ~2 min să pornească (status Running).
  4. Din pagina serviciului, secțiunea Connection information, notează: Host, Port, User, Password și Database name (de obicei defaultdb).

Vei construi din ele URL-ul JDBC (îl folosești la pasul 3):

jdbc:mysql://HOST:PORT/defaultdb?ssl-mode=REQUIRED

Tabelele (utilizatori, roluri, produse, comenzi) se creează automat la prima pornire a backend-ului. Nu trebuie să rulezi manual scripturile SQL.

3 Urcă backend-ul pe Render

  1. Pe render.com → New → Web Service → conectează repo-ul GitHub electroshop.
  2. Setări: Root Directory = backend, Runtime = Docker, Plan = Free.
  3. La Environment Variables adaugă exact:
Nume variabilăValoare
SPRING_DATASOURCE_URLjdbc:mysql://HOST:PORT/defaultdb?ssl-mode=REQUIRED
SPRING_DATASOURCE_USERNAME(User de la Aiven)
SPRING_DATASOURCE_PASSWORD(Password de la Aiven)
JWT_SECRETo1GUewhFu9FvyMzTmRP1kANSP0WP30+9jtQcoS+guI2Ftbh4pR/Ii+QwdiedH14J
CORS_ORIGINShttps://TEMP (îl actualizezi la pasul 5)
  1. Create Web Service. Primul build durează câteva minute.
  2. Când e gata, testează în browser: https://<numele-serviciului>.onrender.com/api/health → trebuie să vezi {"success":true,...,"status":"UP"}.
  3. Notează adresa backend-ului: https://<numele-serviciului>.onrender.com
Există și fișierul render.yaml în proiect — dacă alegi New → Blueprint, Render preia automat aceste setări și îți cere doar valorile.

4 Urcă frontend-ul pe Vercel

  1. Pe vercel.com → Add New → Project → importă repo-ul electroshop.
  2. Setări: Root Directory = frontend. Framework: Vite (detectat automat).
  3. La Environment Variables adaugă:
NumeValoare
VITE_API_URLhttps://<numele-serviciului>.onrender.com/api
  1. Deploy. La final primești adresa site-ului: https://<ceva>.vercel.app

5 Leagă-le și testează

  1. Copiază adresa de la Vercel (ex. https://electroshop.vercel.app).
  2. Înapoi pe Render → serviciul backend → Environment → editează CORS_ORIGINS = adresa Vercel (fără / la final) → Save (redeployează singur).
  3. Deschide site-ul Vercel. Gata — e live!

Contul de administrator este creat automat:

Loghează-te ca admin → meniul Admin → Utilizatori: de acolo setezi rolul (ADMIN/USER) oricărui cont, inclusiv contului tău personal după ce te înregistrezi. Schimbă parola de admin imediat ce intri (din management utilizatori).

⚠️ De reținut pe planul gratuit

  • Pornire „la rece": backend-ul Render adoarme după 15 min fără trafic; prima cerere după pauză durează ~30–60 sec. Normal pe gratuit.
  • Imagini produse: încărcarea prin fișier este temporară pe Render (se pierde la redeploy). Pentru început, folosește câmpul URL imagine din formularul de produs (imagini găzduite extern). Modulul de contabilitate/stoc nu e afectat.
  • Aiven oprește baza după inactivitate lungă, dar datele rămân și repornește la următoarea conectare.
  • Securitate: secretul JWT din acest ghid e doar pentru tine — nu-l publica. Poți genera altul oricând.

➡️ Pasul următor (după ce e live)

Modulul de contabilitate primară: adaug furnizori + intrări de marfă (cumpărări) și un raport cu venituri (vânzări), cheltuieli (cumpărări) și profit/marjă — peste aceeași bază de date. Se adaugă fără să pierzi datele existente.
